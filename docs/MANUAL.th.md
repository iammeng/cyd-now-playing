# Spotify CYD — คู่มือการใช้งาน

🇬🇧 [English](MANUAL.md) · การติดตั้งดู[คู่มือติดตั้ง](SETUP.th.md)

รวมทุกอย่างเกี่ยวกับ*การใช้งาน*จอในชีวิตประจำวัน อ้างอิง firmware **v1.1.1**

---

## 1. หน้าจอต่างๆ

| หน้าจอ | เมื่อไหร่ | เห็นอะไร |
|---|---|---|
| **Boot** | เปิดเครื่อง | โลโก้ + เลขเวอร์ชัน firmware (ที่เดียวที่แสดงเวอร์ชัน) + สถานะ WiFi |
| **Now Playing** | มีเพลงเล่น/หยุดชั่วคราว | ปก ชื่อเพลง ศิลปิน อัลบั้ม ปุ่ม แถบ progress |
| **นาฬิกา** | ต่อได้แต่ไม่มีเพลงเล่น | เวลาปัจจุบัน + โลโก้ Spotify |
| **Offline** | ต่อ server ไม่ได้ | ที่อยู่ server ที่พยายามต่อ + คำแนะนำ |
| **ตั้งค่า WiFi** | ไม่มี WiFi ที่รู้จัก | วิธีเข้า setup portal (ข้อ 6) |

**จุดสถานะ** (มุมขวาบน 3px):
- 🔴 แดง — WiFi หลุด (บอร์ดจะพยายามต่อใหม่เอง)
- 🟡 เหลือง — WiFi ปกติแต่ server ไม่ตอบ
- มองไม่เห็น — ทุกอย่างปกติ (จุดกลืนกับพื้นหลัง)

## 2. การควบคุมด้วยทัช (หน้า Now Playing)

```
┌──────────────────────────────────────────┐
│ ┌────────┐   ชื่อเพลง (สูงสุด 2 บรรทัด)      │
│ │  ปก    │   ศิลปิน                        │   ← โซนนี้ยังไม่มี action (จองไว้)
│ │ อัลบั้ม  │   อัลบั้ม                        │
│ └────────┘                               │
├──────────────────────────────────────────┤
│   🔀      ⏮       ⏯       ⏭       🔁     │   ← แถวปุ่ม: ทัช 5 โซน
├──────────────────────────────────────────┤
│ 0:42  ━━━━━●─────────────────────  3:35  │   ← แตะบนแถบ = seek
└──────────────────────────────────────────┘
        ↕ ปัดขึ้น/ลงที่ไหนก็ได้ = ปรับเสียง
```

| ท่า | ผล |
|---|---|
| แตะ **สลับสุ่ม** (ล่างซ้าย) | เปิด/ปิด shuffle — ไอคอนติดสีธีมอัลบั้มเมื่อเปิด |
| แตะ **ก่อนหน้า** | เพลงก่อนหน้า |
| แตะ **เล่น/หยุด** (วงกลมกลาง) | play / pause |
| แตะ **ถัดไป** | เพลงถัดไป |
| แตะ **เล่นซ้ำ** (ล่างขวา) | วนโหมด: ปิด → ซ้ำทั้งลิสต์ → ซ้ำเพลงเดียว (มีเลข "1") |
| แตะ **แถบ progress** | กระโดดไปตำแหน่งนั้นของเพลง |
| **ปัดขึ้น** (ที่ไหนก็ได้) | เสียง +10% |
| **ปัดลง** (ที่ไหนก็ได้) | เสียง −10% |

- มีวงแหวนขาวแว้บรอบปุ่มยืนยันว่ารับทัชแล้ว
- ตอนปรับเสียง **แถบ volume สีขาว**จะโชว์แทนแถบ progress ~1.5 วิ พร้อมเปอร์เซ็นต์ขวามือ
- บน**หน้านาฬิกา**ควบคุมแบบง่าย: แตะจอส่วนซ้าย/กลาง/ขวา = ก่อนหน้า / เล่น-หยุด / ถัดไป

### เงื่อนไขและข้อจำกัด

- คำสั่งควบคุมต้องมี **Spotify Premium** และมี *active device* (มีแอป Spotify ที่ไหนสักที่กำลังเล่น/pause อยู่) — ถ้ากดแล้วเงียบ ให้เปิดเพลงจากแอปก่อน
- **Volume** สั่งไม่ได้กับอุปกรณ์บางชนิด — ที่เจอชัดคือ iPhone เป็นตัวเล่น ส่วน desktop และลำโพงสั่งได้
- หลังสั่งคำสั่ง จอจะ refresh สถานะจริงจาก Spotify ภายใน ~1 วินาที

## 3. ความสว่างหน้าจอ

Backlight ปรับตามแสงในห้องอัตโนมัติด้วยเซ็นเซอร์แสงบนบอร์ด: ห้องสว่าง → จอสว่างเต็ม, ห้องมืด → จอหรี่

- **แตะจอตอนหรี่** = ปลุกให้สว่างเต็ม **5 นาที**
- การแตะ*ครั้งแรก*ตอนหรี่**แค่ปลุกจอเท่านั้น** — ไม่นับเป็นการกดปุ่ม จึงไม่มีทางเปลี่ยนเพลงพลาดตอนมืด แตะครั้งถัดไปทำงานปกติ (และต่ออายุ 5 นาทีทุกครั้ง)
- ถ้าจอไม่ยอมหรี่ (หรือหรี่ตลอด) แปลว่า threshold ไม่ตรงกับห้อง — ดูค่าจริงที่ `GET /ldr` และตารางปรับแต่งใน[คู่มือติดตั้ง ข้อ 9](SETUP.th.md)

## 4. URL ที่มีประโยชน์ (เปิดจาก browser เครื่องไหนก็ได้ใน LAN)

แทน `<board>` ด้วย IP ของบอร์ด (หรือ `spotify-cyd.local` ถ้า mDNS ใช้ได้)

| URL | ทำอะไร |
|---|---|
| `http://<board>/screen` | screenshot สด (raw dump — ใช้ `tools/capture.py` แปลงเป็น PNG) |
| `http://<board>/flip` | พลิกจอ 180° (เรื่องมุมมองจอ TN) — รีบูต |
| `http://<board>/server?h=<ip>` | ชี้บอร์ดไป server อื่น — รีบูต |
| `http://<board>/server` | ดูว่าใช้ server ไหนอยู่ |
| `http://<board>/test` | หน้าทดสอบจอ 2 นาที |
| `http://<board>/ldr` | ค่าเซ็นเซอร์แสง + ระดับ backlight ปัจจุบัน |
| `http://<board>/touch` | ทัชครั้งล่าสุด (ไว้ calibrate) |

ถ่าย screenshot สวยๆ จากคอม:

```bash
python3 tools/capture.py <ip บอร์ด> screenshot.png
```

## 5. อัปเดต firmware

ผ่าน WiFi (OTA) ไม่ต้องเสียบสาย:

```bash
cd firmware
pio run -e cyd_ota -t upload --upload-port <ip บอร์ด>
```

จอจะขึ้น "OTA update..." ระหว่าง flash แล้วรีบูตเอง เช็คเลขเวอร์ชันใหม่ได้ที่หน้าบูต — ถ้าที่บ้านมีบอร์ด ESP32 หลายตัว **เช็คก่อนว่า IP นั้นเป็นบอร์ดตัวนี้จริง** (เช่น `curl http://<ip>/ldr` — มีแต่โปรเจกต์นี้ที่ตอบ)

## 6. ตั้งค่า WiFi / ย้ายบ้าน

1. บอร์ดจำ WiFi ไว้ข้ามการรีบูต*และ*ข้ามการ flash ใหม่
2. ถ้าต่อไม่ได้ (~1.5 นาทีของการ retry) จะเปิด AP ของตัวเอง:
   - เอามือถือต่อ WiFi ชื่อ **`Spotify-CYD-Setup`**
   - เปิด `192.168.4.1`
   - เลือก WiFi บ้าน ใส่รหัสผ่าน
   - (แก้ **Server IP** ได้ในหน้าเดียวกัน) → Save
3. บอร์ดรีบูตแล้วต่อเอง

ถ้าไม่มีใครตั้งค่าภายใน 3 นาที บอร์ดจะรีบูตแล้วลองใหม่เอง

## 7. แก้ปัญหาด่วน

| อาการ | ลองทำ |
|---|---|
| หน้าจอ "เชื่อมต่อ server ไม่ได้" | เครื่อง server เปิดอยู่ไหม? `docker ps` — แล้วเช็คว่า IP บนจอตรงกับ server |
| ขึ้นนาฬิกาทั้งที่เพลงเล่นอยู่ | token อาจต้อง auth ใหม่ — `docker logs spotify-cyd` ถ้าขึ้น *run auth.py* ให้ auth ใหม่ (คู่มือติดตั้ง ข้อ 5.4) |
| กดปุ่มแล้วเงียบ | เปิดเพลงจากแอป Spotify ก่อน (ต้องมี active device + Premium) |
| ปัดปรับเสียงไม่ทำงาน | อุปกรณ์ตัวเล่นไม่รับคำสั่ง volume (iPhone เป็นแบบนั้น) — เปลี่ยนตัวเล่น |
| ปกไม่ขึ้นสักที | ดู log ของ server (`docker logs -f spotify-cyd`) |
| ถ่ายรูปจอแล้วสีเพี้ยน | moiré ของกล้อง — ไม่ใช่ปัญหาจริง |
| มองเฉียงแล้วจอซีด | เป็นจอ TN — หันจอเข้าหาตัว หรือเปิด `/flip` |
